## إغلاق ثغرة المصادقة

السبب الجذري لم يكن قبول API لأي كلمة مرور؛ API كان يرفض الكلمة الخاطئة بالفعل. الخلل كان أن صفحة SPA تعرض مسارات `/dashboard` دون تحقق Server-side، بينما اختبار E2E القديم اعتمد بيانات دخول محلية ثابتة ولم يفحص قاعدة البيانات.

تم تنفيذ الآتي:

- حماية كل مسارات Dashboard في Server Component باستخدام جلسة قاعدة البيانات.
- إزالة `localDemoLogin` وعدم السماح بأي fallback عند تعطل API.
- التحقق من `/api/auth/session` بعد نجاح login وقبل الانتقال للوحة.
- إبقاء تجاوز واجهة E2E خلف `E2E_UI_PREVIEW=1` في مشغل الاختبار فقط؛ المتغير غير موجود في Compose أو بيئة Staging/Production ولا يتجاوز تحقق كلمة المرور.
- استخدام `scrypt` مع `timingSafeEqual` والفشل المغلق عند hash مفقود أو تالف.
- إضافة `email_hash` إلى سجل محاولات الدخول دون تسجيل كلمة المرور أو hash كلمة المرور أو session token.
- حذف جميع الجلسات السابقة عبر migration جديدة.
